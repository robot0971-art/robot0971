# 크리에이티브 갤러리 게시판 기획안

**작성일:** 2026-03-31  
**프로젝트명:** Creative Gallery Board  
**버전:** 1.0

---

## 1. 프로젝트 개요

### 1.1 목적
크리에이티브한 사람들이 자신의 시각 작품을 자유롭게 공유하고, 
서로 피드백을 주고받을 수 있는 익명 기반 커뮤니티 플랫폼

### 1.2 핵심 가치
- **자유로운 표현:** 익명성을 통해 부담 없는 작품 공유
- **시각적 중심:** 갤러리 형태의 직관적인 작품 탐색
- **건설적 피드백:** 크리에이티브 커뮤니티 특화된 피드백 문화

---

## 2. 사용자 스토리

### 2.1 주요 사용자
1. **작품 공유자** - 자신의 작업물을 업로드하고 피드백을 받고 싶은 사용자
2. **피드백 제공자** - 다른 사람의 작품을 감상하고 의견을 나누고 싶은 사용자
3. **탐색자** - 영감을 얻기 위해 다양한 작품을 둘러보는 사용자

### 2.2 사용자 시나리오
- **시나리오 1:** 사용자 A가 방금 완성한 일러스트를 업로드하고, 다른 사용자들의 반응과 조언을 받는다
- **시나리오 2:** 사용자 B가 갤러리를 둘러보다 마음에 드는 작품을 발견하고, 구체적인 피드백을 남긴다
- **시나리오 3:** 사용자 C가 특정 장르(예: 디지털 아트)의 작품만 필터링하여 탐색한다

## 3. 기능 명세

### 3.1 핵심 기능

#### 3.1.1 갤러리 뷰 (메인 페이지)
- **그리드 레이아웃:** 반응형 그리드 (데스크톱: 4열, 태블릿: 3열, 모바일: 2열)
- **썸네일 표시:** 작품의 썸네일 이미지, 제목, 업로드 시간, 태그
- **무한 스크롤:** 스크롤 시 자동으로 다음 작품들 로드
- **정렬 옵션:** 최신순, 인기순(댓글 수), 랜덤

#### 3.1.2 작품 업로드
- **파일 업로드:** 이미지 파일(JPG, PNG, GIF, WebP) 지원
- **드래그 앤 드롭:** 직관적인 파일 업로드 경험
- **메타데이터 입력:**
  - 제목 (필수, 최대 100자)
  - 설명 (선택, 최대 1000자)
  - 태그 (선택, 최대 5개, 각 20자)
  - 사용 도구/프로그램 (선택)
- **자동 썸네일 생성:** 업로드된 이미지에서 썸네일 자동 생성

#### 3.1.3 작품 상세 보기
- **원본 이미지 표시:** 고해상도 이미지 뷰어
- **작품 정보:** 제목, 설명, 태그, 업로드 시간, 조회수
- **댓글 시스템:**
  - 익명 댓글 작성
  - 댓글에 대한 답글 (스레드 형태)
  - 댓글 정렬 (최신순/인기순)

#### 3.1.4 검색 및 필터링
- **태그 기반 필터링:** 여러 태그 동시 선택 가능
- **키워드 검색:** 제목과 설명 내용 검색
- **장르별 필터:** 미리 정의된 카테고리 (예: 일러스트, 사진, 그래픽 디자인, 3D 등)

### 3.2 추가 기능 (향후 고려)
- 좋아요/하트 시스템
- 작품 컬렉션/북마크
- 인기 작가/작품 랭킹
- 다크 모드 지원

## 4. 기술 아키텍처

### 4.1 기술 스택
- **Frontend:** HTML5, CSS3, Vanilla JavaScript (ES6+)
- **Backend:** Node.js + Express.js
- **Database:** SQLite (파일 기반)
- **Storage:** 로컬 파일 시스템
- **이미지 처리:** Sharp.js (썸네일 생성)

### 4.3 데이터 모델

#### Posts (게시물)
```sql
CREATE TABLE posts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  description TEXT,
  image_path TEXT NOT NULL,
  thumbnail_path TEXT NOT NULL,
  tags TEXT, -- JSON array로 저장
  tools TEXT,
  views INTEGER DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### Comments (댓글)
```sql
CREATE TABLE comments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  post_id INTEGER NOT NULL,
  parent_id INTEGER, -- 답글인 경우 부모 댓글 ID
  content TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (post_id) REFERENCES posts(id),
  FOREIGN KEY (parent_id) REFERENCES comments(id)
);
```

## 6. API 명세

### 6.1 게시물 API

#### GET /api/posts
- **설명:** 게시물 목록 조회 (페이지네이션)
- **파라미터:**
  - `page`: 페이지 번호 (기본값: 1)
  - `limit`: 페이지당 항목 수 (기본값: 20)
  - `sort`: 정렬 기준 (newest|popular|random)
  - `tag`: 태그 필터
- **응답:** 게시물 배열

#### POST /api/posts
- **설명:** 새 게시물 업로드
- **본문:** multipart/form-data
  - `image`: 이미지 파일
  - `title`: 제목
  - `description`: 설명
  - `tags`: 태그 배열 (JSON 문자열)
- **응답:** 생성된 게시물 정보

#### GET /api/posts/:id
- **설명:** 특정 게시물 상세 조회
- **응답:** 게시물 상세 정보 + 댓글 목록

### 6.2 댓글 API

#### POST /api/posts/:id/comments
- **설명:** 댓글 작성
- **본문:**
  - `content`: 댓글 내용
  - `parentId`: 답글인 경우 부모 댓글 ID (선택)
- **응답:** 생성된 댓글 정보

#### GET /api/posts/:id/comments
- **설명:** 특정 게시물의 댓글 목록
- **파라미터:**
  - `sort`: 정렬 기준 (newest|popular)

## 7. 제약사항 및 고려사항

### 7.1 기술적 제약
- **파일 크기:** 업로드 이미지 최대 10MB
- **이미지 형식:** JPG, PNG, GIF, WebP만 지원
- **저장 용량:** 서버 디스크 공간 한계

### 7.2 보안 고려사항
- **파일 업로드 검증:** MIME 타입 및 확장자 검증
- **XSS 방지:** 사용자 입력값 sanitization
- **파일 접근 제어:** 업로드된 파일 직접 접근 제한

### 7.3 성능 고려사항
- **썸네일 최적화:** 원본 이미지와 분리된 작은 크기 썸네일
- **지연 로딩:** 화면에 보이는 이미지만 로드
- **캐싱:** 정적 파일 브라우저 캐싱

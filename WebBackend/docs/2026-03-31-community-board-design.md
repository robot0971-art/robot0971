# 커뮤니티 라운지 (Community Lounge) 기획안

**작성일:** 2026-03-31  
**프로젝트명:** Community Lounge  
**버전:** 1.0

---

## 1. 프로젝트 개요

### 1.1 목적
크리에이티브하고 화려한 디자인의 커뮤니티 게시판 서비스. 사용자는 회원가입 후 로그인하여 게시글을 작성하고, 다른 사용자의 글을 읽고, 댓글과 대댓글로 소통할 수 있는 플랫폼

### 1.2 핵심 가치
- **크리에이티브한 경험:** 네온 톤과 글라스모피즘으로 차별화된 비주얼
- **익숙함:** 누구나 쉽게 사용할 수 있는 일반적인 게시판 UX
- **명확한 권한:** 작성자 중심의 CRUD 권한 관리
- **계층적 소통:** 게시글 → 댓글 → 대댓글의 자연스러운 흐름

---

## 2. 사용자 스토리

### 2.1 주요 사용자
1. **비회원** - 게시글을 읽기만 하는 사용자
2. **회원** - 게시글 작성, 댓글/대댓글 작성이 가능한 사용자
3. **작성자** - 자신의 게시글/댓글을 수정/삭제할 수 있는 사용자

### 2.2 사용자 시나리오
- **시나리오 1:** 비회원 A가 게시판 목록을 둘러보고 글을 읽는다. 댓글을 남기려고 하면 로그인 안내를 받는다.
- **시나리오 2:** 회원 B가 회원가입 후 로그인하고, 새로운 게시글을 작성한다.
- **시나리오 3:** 회원 C가 다른 사용자의 글에 댓글을 남기고, 댓글에 대한 대댓글을 받는다.
- **시나리오 4:** 작성자 D가 자신의 글에 오타를 발견하고 수정한다. 필요 없어진 댓글은 삭제한다.

## 3. 기능 명세

### 3.1 회원 기능

#### 3.1.1 회원가입
- **입력 항목:**
  - 사용자 ID (필수, 영문/숫자, 4-20자, 중복 불가)
  - 비밀번호 (필수, 6-20자)
  - 비밀번호 확인 (필수)
  - 닉네임 (필수, 2-10자, 중복 불가)
- **유효성 검사:**
  - 필수 항목 누락 시 "필수 정보를 입력해주세요"
  - ID 중복 시 "이미 사용 중인 아이디입니다"
  - 닉네임 중복 시 "이미 사용 중인 닉네임입니다"
  - 비밀번호 불일치 시 "비밀번호가 일치하지 않습니다"
- **가입 완료:** 로그인 페이지로 리다이렉트 + "회원가입이 완료되었습니다" 메시지

#### 3.1.2 로그인
- **입력 항목:**
  - 사용자 ID
  - 비밀번호
- **로그인 성공:**
  - 세션 생성
  - 메인 페이지(게시판 목록)로 리다이렉트
  - "환영합니다, {닉네임}님" 메시지
- **로그인 실패:**
  - "아이디 또는 비밀번호가 올바르지 않습니다" 메시지
  - 입력값 유지 (비밀번호 제외)
- **세션 유지:** 브라우저 종료 전까지 로그인 상태 유지

#### 3.1.3 로그아웃
- **동작:** 세션 삭제, 로그인 상태 해제
- **완료 후:** 메인 페이지로 리다이렉트 + "로그아웃되었습니다" 메시지

#### 3.1.4 로그인 상태 표시
- **헤더 영역에 표시:**
  - 로그인 전: "로그인" | "회원가입" 링크
  - 로그인 후: "{닉네임}님" | "로그아웃" | "글쓰기" 링크

### 3.2 게시판 기본 기능

#### 3.2.1 게시글 목록
- **표시 항목:**
  - 글 번호 (자동 생성)
  - 제목
  - 작성자 (닉네임)
  - 작성일 (YYYY-MM-DD HH:mm 형식)
  - 댓글 수 (댓글+대댓글 합계)
- **정렬:** 최신순 (created_at DESC)
- **페이지네이션:** 페이지당 10개, 페이지 번호 네비게이션
- **작성자 표시:** 탈퇴한 회원은 "알 수 없음"으로 표시

#### 3.2.2 게시글 상세 조회
- **표시 내용:**
  - 제목
  - 작성자 (닉네임)
  - 작성일
  - 본문 내용
  - 댓글 목록 (하단)
- **작성자용 버튼:** 수정 | 삭제 (작성자 본인만 표시)
- **댓글 작성 폼:** 로그인한 사용자에게만 표시
- **비회원:** "댓글을 작성하려면 로그인이 필요합니다" 안내

#### 3.2.3 존재하지 않는 게시글 접근
- **에러 메시지:** "존재하지 않는 게시글입니다"
- **안내:** "게시판 목록으로 돌아가기" 버튼

### 3.3 게시글 작성, 수정, 삭제

#### 3.3.1 게시글 작성
- **권한:** 로그인한 회원만 가능
- **입력 항목:**
  - 제목 (필수, 최대 100자)
  - 내용 (필수, 최대 5000자)
- **유효성 검사:**
  - 제목 누락 시 "제목을 입력해주세요"
  - 내용 누락 시 "내용을 입력해주세요"
- **작성 완료:** 게시글 상세 페이지로 리다이렉트

#### 3.3.2 게시글 수정
- **권한:** 작성자 본인만 가능
- **화면:** 기존 내용이 입력된 상태로 표시
- **수정 가능 항목:** 제목, 내용
- **수정 완료:** 게시글 상세 페이지로 리다이렉트 + "수정되었습니다" 메시지
- **권한 없음:** "작성자만 수정할 수 있습니다" 메시지

#### 3.3.3 게시글 삭제
- **권한:** 작성자 본인만 가능
- **확인:** "정말 삭제하시겠습니까?" 확인 대화상자
- **삭제 완료:** 게시판 목록으로 리다이렉트 + "삭제되었습니다" 메시지
- **삭제 후:** 해당 게시글은 비회원/회원 모두 조회 불가
- **권한 없음:** "작성자만 삭제할 수 있습니다" 메시지

### 3.4 댓글 기능

#### 3.4.1 댓글 작성
- **권한:** 로그인한 회원만 가능
- **입력:** 내용 (최대 1000자)
- **작성 위치:** 게시글 상세 페이지 하단
- **작성 완료:** 페이지 새로고침 후 댓글 목록에 추가
- **비회원:** "댓글을 작성하려면 로그인이 필요합니다" 안내 링크

#### 3.4.2 댓글 수정
- **권한:** 작성자 본인만 가능
- **동작:** 해당 댓글 영역이 입력 폼으로 변경 → 저장 시 페이지 새로고침
- **수정 완료:** 페이지 새로고침 후 반영
- **취소:** 수정 취소 버튼으로 원래 내용 복원
- **권한 없음:** "작성자만 수정할 수 있습니다" 메시지

#### 3.4.3 댓글 삭제
- **권한:** 작성자 본인만 가능
- **확인:** "정말 삭제하시겠습니까?" 확인 대화상자
- **삭제 완료:** 페이지 새로고침, 댓글 목록에서 제거
- **대댓글 있음:** 원댓글 삭제 시 "삭제된 댓글입니다"로 표시, 대댓글은 유지
- **권한 없음:** "작성자만 삭제할 수 있습니다" 메시지

### 3.5 대댓글 기능

#### 3.5.1 대댓글 작성
- **권한:** 로그인한 회원만 가능
- **작성 위치:** 특정 댓글의 "답글" 버튼 클릭 시 입력 폼 표시
- **표시 방식:** 원댓글 아래에 들여쓰기 또는 구분선으로 표시
- **관계 표시:** "@{원댓글작성자}" 형태로 참조 표시 (선택사항)
- **제한:** 대댓글의 대댓글은 지원하지 않음 (1depth만 지원)

#### 3.5.2 대댓글 수정/삭제
- **권한:** 작성자 본인만 가능
- **동작:** 일반 댓글과 동일

#### 3.5.3 댓글 계층 표시
```
┌─ [글라스 댓글 카드 #1] 작성자A: 좋은 글이네요
│  작성일: 2024-01-01
│  └─ [대댓글 글라스 카드] 작성자B: 감사합니다
│     (들여쓰기: 24px + 왼쪽 네온 보라 라인)
│
├─ [글라스 댓글 카드 #2] 작성자C: 궁금한 점이 있어요
│  └─ [대댓글] 작성자A: 어떤 점이 궁금하신가요?
│
└─ [글라스 댓글 카드 #3] 작성자D: 공유 감사합니다
```

**스타일:**
- **원댓글:** 글라스 카드 + 네온 핑크 보더 (왼쪽 3px)
- **대댓글:** 글라스 카드 + 네온 보라 보더 (왼쪽 3px) + 24px 들여쓰기
- **작성자명:** 네온 핑크 강조
- **작성일:** 작은 텍스트, 보조 색상
- **버튼:** 수정(네온 블루), 삭제(네온 레드), 답글(네온 보라)
- **호버 효과:** 카드에 네온 글로우 추가

## 4. 기술 아키텍처

### 4.1 기술 스택
- **Frontend:** HTML5, CSS3, Vanilla JavaScript
- **Backend:** Node.js + Express.js
- **Database:** SQLite3
- **Session:** express-session (MemoryStore 또는 파일 기반)
- **템플릿 엔진:** EJS

### 4.3 데이터 모델

#### Users (회원)
```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT UNIQUE NOT NULL,      -- 로그인 ID
  password TEXT NOT NULL,             -- 해시된 비밀번호
  nickname TEXT UNIQUE NOT NULL,      -- 표시명
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### Posts (게시글)
```sql
CREATE TABLE posts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  view_count INTEGER DEFAULT 0,       -- 조회수
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL
);
```

#### Comments (댓글/대댓글)
```sql
CREATE TABLE comments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  post_id INTEGER NOT NULL,
  user_id INTEGER NOT NULL,
  parent_id INTEGER,                  -- NULL이면 원댓글, 값 있으면 대댓글
  content TEXT NOT NULL,
  is_deleted BOOLEAN DEFAULT 0,       -- 삭제된 댓글 표시용
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (post_id) REFERENCES posts(id) ON DELETE CASCADE,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL,
  FOREIGN KEY (parent_id) REFERENCES comments(id) ON DELETE CASCADE
);
```

### 4.4 API 엔드포인트

#### 인증 API
- `POST /auth/register` - 회원가입
- `POST /auth/login` - 로그인
- `POST /auth/logout` - 로그아웃
- `GET /auth/check-username?username=xxx` - ID 중복 확인
- `GET /auth/check-nickname?nickname=xxx` - 닉네임 중복 확인

#### 게시글 API
- `GET /posts` - 게시글 목록 (페이지네이션)
- `GET /posts/:id` - 게시글 상세
- `POST /posts` - 게시글 작성 (회원전용)
- `PUT /posts/:id` - 게시글 수정 (작성자만)
- `DELETE /posts/:id` - 게시글 삭제 (작성자만)

#### 댓글 API
- `POST /posts/:id/comments` - 댓글 작성 (회원전용)
- `PUT /comments/:id` - 댓글 수정 (작성자만)
- `DELETE /comments/:id` - 댓글 삭제 (작성자만)
- `POST /comments/:id/reply` - 대댓글 작성 (회원전용)

## 5. 화면 설계

### 5.1 공통 레이아웃
- **배경:** 딥 블루/퍼플 그라데이션 (`#1a1a2e` → `#16213e` → `#0f3460`)
- **헤더:** 
  - 글라스모피즘 스타일 (반투명 + 블러)
  - 좌측: **Community Lounge** 네온 스타일 로고 (네온 핑크/보라 그라데이션)
  - 우측: 로그인/회원가입(네온 핑크) 또는 닉네임/로그아웃/글쓰기(네온 블루)
  - 하단 네온 보더 라인
- **본문:** 페이지별 콘텐츠 (글라스 카드들로 구성)
- **푸터:** 반투명 다크 + 네온 텍스트 "© 2026 Community Lounge"

### 5.2 주요 화면

#### 5.2.1 게시글 목록 (`/`)
- **글라스 테이블 카드:** 반투명 배경 + 네온 보더
- **테이블 헤더:** 그라데이션 배경 (`#ff006e` → `#9d4edd`)
- **행 구분:** 홀수/짝수 행 미세한 색상 차이 + 호버 시 네온 글로우
- **테이블 컬럼:** 번호 | 제목 | 작성자 | 작성일 | 댓글수
- **제목 링크:** 네온 블루(`#00d9ff`) + 호버 시 네온 글로우
- **작성자:** 네온 핑크(`#ff006e`) 강조
- **댓글 수 배지:** 둥근 모서리 + 네온 보라 배경
- **하단:** 네온 스타일 페이지네이션
- **우측 상단:** "글쓰기" 버튼 (그라데이션 + 네온 그림자, 로그인 시)

#### 5.2.2 게시글 상세 (`/posts/:id`)
- **글라스 카드:** 게시글 본문 영역
- **제목:** 대형 폰트 + 네온 핑크 그림자 효과
- **메타 정보:** 작성자(네온 핑크) | 작성일(보조 텍스트) | 조회수(네온 블루 아이콘)
- **본문 내용:** 흰색 텍스트 + 글라스 패딩 박스
- **작성자용 버튼:** 
  - 수정: 네온 블루 버튼
  - 삭제: 네온 레드 버튼
- **댓글 섹션:**
  - 구분선: 그라데이션 라인
  - 댓글 목록: 계층형 + 들여쓰기 라인(네온 보라)
  - 댓글 작성 폼: 글라스 인풋 + 네온 포커스 효과 (로그인 시)

#### 5.2.3 게시글 작성/수정 (`/posts/new`, `/posts/:id/edit`)
- **글라스 카드 컨테이너:** 반투명 배경 + 네온 보더
- **제목 입력:** 다크 인풋 + 네온 블루 포커스 효과
- **내용 텍스트 영역:** 다크 배경 + 스크롤바 네온 스타일
- **저장 버튼:** 그라데이션(핑크→보라) + 네온 그림자
- **취소 버튼:** 반투명 + 보더

#### 5.2.4 로그인 (`/login`)
- **글라스 로그인 카드:** 중앙 정렬 + 네온 글로우 효과
- **로고:** **Community Lounge** 네온 스타일 (핑크→보라 그라데이션)
- **서브타이틀:** "크리에이티브한 사람들의 라운지" (네온 블루)
- **ID 입력:** 다크 인풋 + 네온 블루 포커스
- **비밀번호 입력:** 다크 인풋 + 마스킹
- **로그인 버튼:** 그라데이션(핑크→보라) + 호버 시 강렬한 글로우
- **"회원가입" 링크:** 네온 블루 + 밑줄 애니메이션

#### 5.2.5 회원가입 (`/register`)
- **글라스 카드:** 중앙 정렬 + 네온 보더
- **ID 입력:** 다크 인풋 + 중복확인 버튼(네온 스타일)
- **비밀번호 입력:** 강도 표시(네온 색상으로 약/중/강)
- **비밀번호 확인:** 일치 시 네온 그린, 불일치 시 네온 레드
- **닉네임 입력:** 다크 인풋 + 중복확인 버튼
- **가입 버튼:** 그라데이션 + 네온 효과
- **유효성 메시지:** 네온 레드(오류) / 네온 그린(성공)

### 5.3 색상 및 스타일 (크리에이티브 네온 테마)

**디자인 컨셉:** 크리에이티브 아트 갤러리 - 화려한 네온 톤과 글라스모피즘

#### 기본 색상 팔레트
- **페이지 배경 그라데이션:** `linear-gradient(135deg, #1a1a2e 0%, #16213e 50%, #0f3460 100%)` (딥 블루/퍼플)
- **카드 배경:** `rgba(255, 255, 255, 0.05)` + `backdrop-filter: blur(10px)` (글라스 효과)
- **텍스트 주요:** `#ffffff` (흰색)
- **텍스트 보조:** `#e0e0e0`, `#a0a0a0` (연한 회색 계열)
- **보더:** `rgba(255, 255, 255, 0.1)` (반투명 흰색)

#### 네온 강조색 (Neon Accent Colors)
- **네온 핑크 (Primary):** `#ff006e` - 주요 버튼, 링크, 활성 상태
- **네온 블루:** `#00d9ff` - 하이라이트, 아이콘, 호버 효과
- **네온 보라:** `#9d4edd` - 태그, 뱃지, 강조 텍스트
- **네온 오렌지:** `#ff8500` - 경고, 알림, 특별 표시
- **네온 그린:** `#00f5d4` - 성공, 저장 완료 표시

#### 버튼 스타일
- **Primary 버튼:** `linear-gradient(135deg, #ff006e, #9d4edd)` + `box-shadow: 0 4px 15px rgba(255, 0, 110, 0.4)`
- **Secondary 버튼:** `rgba(255, 255, 255, 0.1)` + `border: 1px solid rgba(255, 255, 255, 0.2)`
- **위험 버튼:** `linear-gradient(135deg, #ff006e, #ff4d6d)` + 네온 레드 그림자

#### 태그/뱃지 스타일

## 11. 결론

**Community Lounge**는 크리에이티브하고 화려한 디자인의 일반 커뮤니티 게시판을 목표로 합니다. 네온 톤과 글라스모피즘을 활용한 차별화된 비주얼 디자인이 핵심이며, 회원 시스템을 기반으로 하여 게시글과 댓글/대댓글을 통한 소통이 핵심입니다. 명확한 권한 관리와 적절한 예외 처리로 사용자 경험을 해치지 않는 것이 중요합니다.

**핵심 목표:**
- 크리에이티브한 네온/글라스 디자인으로 차별화된 "라운지" 경험 제공
- 누구나 쉽게 사용할 수 있는 직관적인 UI
- 명확한 작성자 중심의 권한 관리
- 게시글 → 댓글 → 대댓글의 자연스러운 흐름

**브랜드 아이덴티티:**
- **이름:** Community Lounge (커뮤니티 라운지)
- **컨셉:** 크리에이티브한 사람들이 모여 편안하게 소통하는 공간
- **슬로건:** "크리에이티브한 사람들의 라운지"

**디자인 키포인트:**
- 딥 블루/퍼플 그라데이션 배경
- 네온 핑크/블루/보라 강조색
- 글라스모피즘 카드와 패널
- 부드러운 네온 글로우 효과
